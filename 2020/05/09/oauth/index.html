What is OAuth

An open protocol to allow secure authorization in a simple and standard method from web, mobile and desktop applications.
– from https://oauth.net/

  • OAuth is a protocol
  • OAuth related to authorization

We will think, hmm… what authorization? What exactly the OAuth provide?

I think the explaction at wiki is more clear.

OAuth is an open standard for access delegation, commonly used as a way for Internet users to grant websites or applications access to their information on other websites but without giving them the passwords

Through OAuth protocol or technology, we, users can authoirize some web/mobile/desktop applicaiton to have our information, like location, gender, age …, without giving passwords to applicaiton providers

OAuth Terminology

Roles

  • Resource Owner: a person that allow the appication can access to the person’s protected data
  • Client: the appication which can represents resource owner to access protected data if resource owner have authorized it
  • Authorization Server (Auth Server): a machine that make sure resource owner indeed has the authorization to access thier own protected data
  • Resource Server: a machine that stores protected data of resource owner

NOTES: auth server and resource server can be the same machine

Other

  • Authorization Grant: the action that resource owner confirm to give the authrization to client applciation
  • Redirect URL: the auth server or resource server need to know where sholud redirect back if the resource owner grant the permission or not
  • Access Token: the string/key/token that client get from auth server, to prove that the client applicaiton indeed has the permission to access the resource owner’s protected data

it might still confusing with these terminolgy, the following example will help us udnerstand easier.

Real world example explanation

Let’s deep dive about OAuth with the use case of spotify scenario (everyone loves music, right?)

Assume you are new user to spotify, and want to sign up an account, you might see this page

a button with sign up with facebook
assume you have facebook account already, so you want to sign up with existed facebook account, then you don’t have to remember one more password related to spotify, awesome, right?

and it will redirect to a page that you might familiar with , login with facebook account

Let’s link the scenario to the terminology

here

  • the user is resource owner
  • spotify is client
  • facebook managed the auth server and resource server, that means facebook has your protected data and can record and know if you authorize facebook to give these protected data to spotify

after you login to facebook, facebook will ask you if you want to grant the permission to spotify, if you click yes it means you grant the authorization

and facebook auth server will give a access token to spotify appllcation

Summary

That’s basically how the OAuth work.
